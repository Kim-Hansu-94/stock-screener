# 주식 스크리너 웹사이트 — 설계 (1단계 MVP)

## 배경 및 목표

매일 아침 시장 상황을 점검하고, "지금 시장을 주도하는 섹터 안에서, 추세는 살아있지만 잠깐 가격이 쉬어가는(눌림목)" 종목을 자동으로 걸러서 보여주는 웹사이트를 만든다. 모바일에서도 보여야 하고, 로그인 없이 링크만으로 지인들과 공유할 수 있어야 한다. 매일 아침 8시 30분(KST)에 자동으로 갱신된다.

이 스펙은 1단계(MVP) 범위만 다룬다. 아래 기능은 별도 단계로 분리한다:

- **2단계**: 에바테크놀로지스(라이다 센서 업체, 7천원→4만원 급등 사례)의 급등 전 차트와 유사한 패턴을 가진 종목을 찾아 장기투자용으로 별도 추천하는 기능. 1단계의 필터링 결과와는 독립적인 별도 리스트.
- **3단계**: 눌림목 필터 기준을 과거 데이터에 적용했을 때의 수익률을 검증하는 백테스팅 기능.

검토했지만 채택하지 않은 아이디어: 국민연금 보유/매수 현황 표시 (DART Open API 연동이 필요해 범위에서 제외).

## 대상 시장 및 종목 범위

- **한국**: KRX 상장 종목 전체 (KOSPI + KOSDAQ), 시가총액 3,000억 원 이상
- **미국**: S&P500 + 나스닥100 구성종목, 시가총액 2억 달러 이상

## 스크리닝 로직

### 1. 시장 분위기 판단 (상승장/하락장)

- 한국은 KOSPI 지수, 미국은 S&P500 지수를 기준으로 각각 독립적으로 판단
- 판단 기준: 지수 종가가 50일 이동평균선 위에 있고, 50일 이동평균선이 200일 이동평균선 위에 있으면 "상승장", 그 외는 "하락장"
- "우상향" 판정 공통 기준: 이동평균선 값이 5거래일 전 값보다 높으면 우상향으로 판단 (이하 60일선 우상향 판단에도 동일하게 적용)
- 하락장으로 판단된 시장은 사이트 상단에 경고 배지를 표시 (예: "현재 하락장입니다, 신중하게 접근하세요")

### 2. 주도섹터 탐지

- 업종 분류: 한국은 KRX 업종분류, 미국은 GICS 섹터(yfinance 제공 데이터)
- 측정: 섹터별 최근 5일 평균 거래액(거래량 × 종가) 합계
- 거래액 상위 섹터 중, 최근 5일간 섹터 평균 가격이 상승(양의 모멘텀)인 섹터만 후보로 인정
- 한국/미국 시장 각각 독립적으로 상위 3개 섹터를 주도섹터로 선정

### 3. 눌림목 매수 필터

주도섹터에 속한 종목 중 아래 조건을 모두 만족하는 종목을 선별한다 (AND 조건):

- 시가총액 조건 충족 (한국 3,000억 원 / 미국 2억 달러 이상)
- **장기 추세**: 60일 이동평균선이 우상향 중이며(위의 공통 기준 적용), 현재가가 60일선 위에 있음
- **단기 눌림**: 현재가가 5일 이동평균선 아래로 내려왔으나, 20일 이동평균선의 ±3% 범위 내 또는 그 위에 있음 (추세가 무너지지 않은 조정)
- **RSI(14일)**: 40~55 구간
- **거래량 감소**: 최근 5일 평균 거래량이 직전 20일 평균 거래량보다 낮음 (조정 중 매도 압력 약화 신호)

## 데이터 파이프라인

- **실행 주기**: 매일 8:30 KST (UTC 23:30, 전날 기준), GitHub Actions 스케줄 워크플로우로 실행
- **데이터 소스**:
  - 한국: pykrx + FinanceDataReader (무료, API 키 불필요) — OHLCV, 시가총액, 업종 분류
  - 미국: yfinance (무료, API 키 불필요) — OHLCV, 시가총액, GICS 섹터
- **처리 과정**:
  1. 한국/미국 전체 대상 종목의 최근 120 거래일 OHLCV 수집
  2. 시장지수 데이터로 시장분위기 계산
  3. 업종별 거래액 집계로 주도섹터 계산
  4. 종목별 기술지표(RSI, 이동평균, 거래량 비율) 계산 후 눌림목 필터 적용
  5. 결과를 Supabase(Postgres)에 저장

## 데이터 저장 (Supabase)

날짜별로 누적 저장하여 추후 히스토리 조회 및 백테스팅(3단계)에 활용한다.

- `market_regime`: 날짜, 시장(KR/US), 상승장 여부
- `leading_sectors`: 날짜, 시장(KR/US), 섹터명, 거래액 순위
- `screened_stocks`: 날짜, 종목코드/티커, 종목명, 시장, 섹터, 현재가, 등락률, 시가총액, RSI, 60일선 대비 위치 등
- `stock_price_history`: 종목코드/티커, 날짜, OHLCV — 필터를 통과한 종목에 대해 최근 120일치를 유지

## 프론트엔드

- **기술 스택**: Next.js + Tailwind CSS, Vercel에 호스팅 (무료 티어)
- **접속**: 로그인 없는 공개 URL. 지인들에게 링크 공유만으로 접속 가능
- **모바일 대응**: 반응형 레이아웃 기본 적용
- **화면 구성**:
  - 상단: 한국/미국 시장분위기 배지 (상승장/하락장)
  - 주도섹터 3개씩 (한국/미국) 표시
  - 눌림목 필터를 통과한 종목 리스트 (카드 형태) — 종목명, 티커, 현재가, 등락률, 시가총액, 소속 섹터, RSI 표시
  - 종목 카드 클릭 시 캔들차트 펼침: 5일/20일/60일 이동평균선 오버레이 + 거래량 바 차트 (lightweight-charts 라이브러리 사용)
