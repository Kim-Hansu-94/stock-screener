# S&P 500 조정 매수 탭 설계

- 날짜: 2026-07-10
- 상태: 설계 승인됨 (구현 전)
- 대상: 김한수의 보물지도에 S&P 500 장기 적립 전략 탭(`/sp500`) 추가

## 목적

S&P 500 추종 ETF를 조정이 올 때마다 분할 매수하는 장기 투자 전략을 지원한다.
탭은 "지금이 조정 몇 단계인지"와 "예비자금 중 얼마를 투입할 차례인지"를 한눈에 보여준다.
관점은 주봉/월봉 장기 투자이므로 전일 종가 기준 하루 1회 갱신이면 충분하다.

## 확정된 결정

| 항목 | 결정 |
|---|---|
| 시그널 기준 | S&P 500 지수(^GSPC) 종가 하나로 계산 |
| 매수 대상 표시 | 국내 상장 ETF(KODEX 미국S&P500TR, TIGER 미국S&P500)와 미국 ETF(SPY, VOO) 둘 다 |
| 조정 정의 | 52주 최고 종가 대비 하락률 단계별 + RSI/이평선은 참고 표시만 |
| 매수 규모 | 단계별 예비자금 비율 제시 (기본 20/30/30/20) |
| 아키텍처 | A안: 파이프라인에서 매일 계산 → Supabase 저장 → 프론트는 표시만 |

## 1. 조정 단계 정의 (시그널 로직)

기준: S&P 500 지수 종가의 52주(252거래일) 최고 종가 대비 하락률.

| 단계 | 조건 | 행동 가이드 | 예비자금 투입 |
|---|---|---|---|
| 0 · 평상 | -5% 미만 | 대기, 현금 적립 | — |
| 1 · 조정 진입 | -5% ~ -10% | 1차 매수 | 20% |
| 2 · 조정 | -10% ~ -20% | 2차 매수 | +30% (누적 50%) |
| 3 · 약세장 | -20% ~ -30% | 3차 매수 | +30% (누적 80%) |
| 4 · 폭락 | -30% 이상 | 전량 투입 | +20% (누적 100%) |

경계값 규칙: 하락률이 정확히 경계에 닿으면 깊은 단계로 판정한다 (-5.0%는 1단계, -10.0%는 2단계).

핵심 규칙:

- **단계 첫 진입 시에만 트리거.** 같은 사이클 안에서 -11% → -9% → -12%로
  왕복해도 2단계 매수 시그널은 최초 1회만 발생한다. 낮은 단계로 되돌아가도
  이미 발생한 단계는 소멸하지 않는다.
- **사이클 리셋.** 지수가 52주 신고가를 갱신하면 사이클이 종료된다.
  모든 단계 기록이 초기화되고 예비자금 적립을 다시 시작한다.
  사이클은 `cycle_id`(신고가 갱신마다 +1)로 식별한다.
- **건너뛴 단계 처리.** 급락으로 1단계를 거치지 않고 2단계에 바로 진입하면
  1·2단계 시그널이 같은 날 함께 발생한 것으로 기록한다 (누적 비율 유지).

보조지표는 매수 조건에 포함하지 않고 참고 표시만 한다:
주봉 RSI(14), 월봉 20이평선 이격도, 200일선 이격도.

## 2. 탭 화면 구성 (`/sp500`)

내비게이션 라벨: `S&P 500 적립`. 위에서 아래 순서:

1. **현재 상태 히어로** — "52주 고점 대비 -X.X%" 게이지, 현재 단계 뱃지,
   한 줄 행동 가이드 (예: "1단계 조정 — 예비자금 20% 매수 구간").
2. **매수 플랜 카드** — 예비자금 금액 입력(localStorage 저장) →
   단계별 투입 금액 표 계산. 현재 사이클에서 이미 트리거된 단계는 체크 표시.
3. **다음 트리거 가격** — 다음 단계 도달 지수와 현재가 대비 남은 거리
   (예: "-10% 도달 지수 5,432 · 현재가에서 -3.2%").
4. **ETF 시세 카드** — 국내 2종(원화)·미국 2종(달러) 종가와 등락률, 기준일 표시.
5. **주봉/월봉 차트** — 지수 주봉 차트에 52주 고점선과 단계별 트리거 가격선
   (-5/-10/-20/-30%)을 겹쳐 그림. 월봉 토글.
6. **보조지표 카드** — 주봉 RSI, 월봉 20선 이격, 200일선 이격.
7. **시그널 이력** — 과거 단계 진입 기록(날짜·단계·당시 지수)과
   그 시점 매수 시 현재까지 수익률.
8. **백테스트 근거** — 정기 적립 대비 이 전략의 역사적 성과 요약 (정적 표시).

## 3. 데이터 파이프라인 & 테이블 스키마

### 파이프라인

새 모듈 `pipeline/src/sp500_monitor.py`를 만들고 일일 파이프라인(`pipeline.py`)
끝에 한 단계로 추가한다.

- ^GSPC 일봉을 yfinance로 수신. 최초 1회 15년치 백필, 이후 매일 증분.
- 52주 고점, 하락률, 단계, 사이클 상태, 주봉 RSI, 월봉 20선, 200일선 계산.
- 국내 ETF 2종은 기존 KIS API 모듈(`prices_kr`) 활용, SPY/VOO는 yfinance.
- Supabase upsert.

`stage` 판정과 사이클 리셋은 파이프라인에서 확정해 저장한다.
프론트는 계산하지 않고 읽기만 한다.

### Supabase 테이블

```
sp500_daily        -- 지수 일봉 + 계산값 (차트·현재상태 소스)
  date PK, close, high_52w, drawdown_pct, stage,
  rsi_w, ma20_m_gap, ma200_d_gap

sp500_signals      -- 단계 진입 이벤트 기록 (이력 소스)
  id PK, date, stage, index_close, drawdown_pct, cycle_id

sp500_etf_quotes   -- ETF 시세 (매일 덮어씀, 최신만 유지)
  ticker PK, name, close, change_pct, currency, updated_at
```

"이번 사이클에서 이미 지나간 단계"는 `sp500_signals`에서 현재 `cycle_id`의
기록을 조회해 판단한다.

### 프론트

`/sp500` 페이지는 기존 페이지 패턴(서버 컴포넌트 + Supabase 조회 + revalidate)을
따른다. 차트는 `sp500_daily` 일봉(15년치 약 3,800행)을 받아 클라이언트에서
주봉/월봉으로 리샘플한다.

## 4. 백테스트 검증

구현 전에 기존 `pipeline/research/backtest_spy_dip.py`를 확장해 검증한다.

1. 단계별 분할 매수(20/30/30/20) vs 정기 적립 vs 단일 임계값 매수 비교.
   기간은 15년 + 가능하면 2000년부터 (닷컴버블·금융위기 포함).
2. 트리거 단계(-5/-10/-20/-30)와 배분 비율 대안의 그리드 비교.

결과 요약은 탭 하단에 정적 표시하고 상세는 `docs/backtest-guide.md`에 기록한다.
결과가 기본 비율과 크게 다르면 비율 상수만 조정한다 (구조 동일).

## 5. 에러 처리 & 테스트

- yfinance 수신 실패: 해당 단계만 건너뛰고 기존 데이터 유지.
  파이프라인 전체는 실패시키지 않는다. 프론트는 항상 기준일을 표시한다.
- ETF 시세 수집 실패: 시그널과 무관하므로 카드에 갱신 실패 표시만.
- pytest 커버 (`pipeline/tests/test_sp500_monitor.py`):
  신고가 갱신 시 사이클 리셋, 단계 왕복 시 중복 트리거 방지,
  경계값(정확히 -5%, -10%) 판정, 급락 시 단계 건너뜀 처리.

## 범위 밖 (YAGNI)

- 실시간/장중 시세 갱신
- 매수 체결 기록·포트폴리오 수익률 추적 (기존 보유 종목 점검 탭과 별개)
- 알림/푸시 발송
- 사용자별 설정 서버 저장 (예비자금은 localStorage로 충분)
